Browser-GLiNER Prototype

Runs urchade/gliner_multi_pii-v1 as ONNX in-browser (GLiNER.js + ONNX Runtime Web), with the verbatim production post-processing + the real regex / allowlist / compound-risk modules, then scores all 120 traps the same way as runTrapsWire.mjs --gliner-only. Answers two questions: (1) does the quantized in-browser model hold the production gate (target 120/120, 0 wire leaks)? (2) is the latency acceptable in this browser — especially Safari?
