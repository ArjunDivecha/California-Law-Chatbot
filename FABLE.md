## 1. Assessment

This is, by a wide margin, the most disciplined app in Arjun's ecosystem, and it is
sound. For a small-firm legal product the server-side trust boundary is real and mostly
fail-closed:

- **Gating defaults to deny.** `guardToolQuery` (`api/_lib/compliance/toolQueryGuard.ts:61-95`) defaults every branch to `allowed:false`; `decidePolicy` (`api/_lib/compliance/policyEngine.ts:163-268`) only permits external calls after consent/role checks; the PII backstop rejects with 503 on any detected raw span (`api/_lib/agentProxy.ts:111-120`).
- **Auth is enforced everywhere it matters.** Every `api/agent/*` route calls `requireUser()` before work (`turn.ts:36`, `turn-stream.ts:51`, `draft-stream.ts:84`, `drafting-magic.ts:114`, `revise-section.ts:56`, `verify-stream.ts:49`, `session.ts:49`, `sessions.ts:39`), and the local `dev-user` bypass is hard-disabled on Vercel/prod (`httpGuard.ts:120-123`).
- **The agent loop is bounded and error-tolerant.** Iteration cap from `agents/california-legal/agent.json` (`max_iterations:12`), loop at `agentLoop.ts:1074/1372`, graceful "max rounds" exit at `:1156-1159`; every tool call is wrapped and returns an `is_error` tool_result rather than throwing (`api/_lib/tools/index.ts:317-324`).
- **Hygiene is high.** Zero `@ts-ignore`/`@ts-expect-error` in the tree, no empty `catch {}` blocks in `api/`/`services/`/`components/`/`hooks/`, model use fenced by a fail-closed allowlist (`api/_lib/approvedModels.ts`), retirements documented in-code rather than silently deleted (`policyEngine.ts:68-71`).

The problems are not in the design — they are in **verification discipline and structural duplication**, and one of them is currently a live confidentiality gap:

- **Live defect (confidentiality):** the on-device wire pipeline no longer tokenizes an in-format California driver-license number. `tsx tests/traps/runTrapsWire.mjs` reports **119/120** with `T-PII-032 missed=driver_license:"A1234567"`. The `CA_DRIVER_LICENSE` regex exists (`api/_shared/sanitization/patterns.ts:58`) and its unit test passes (`tests/sanitization.test.mjs:103`), so the span is being *dropped inside the fuller wire pipeline* (`services/sanitization/detectionPipeline.ts`), not for lack of a pattern. Commit `bc3677f` recorded a prior 120/120 — this is a regression.
- **Root systemic cause:** there is **no CI test gate**. Only `test:sanitization` and `test:traps` are npm scripts; the other 14 `tests/*.test.{mjs,ts}` and `runTrapsWire.mjs` (the runner that catches wire regressions) have no entry point, and the only GitHub Action (`.github/workflows/openwiki-update.yml`) is a docs bot. A confidentiality regression shipped precisely because nothing ran the test that would have caught it.
- **Structural duplication (two of them):** (a) two sanitizer code paths — `analyze()` (`api/_shared/sanitization/index.ts`) and `detectPii()` (`services/sanitization/detectionPipeline.ts`) — share the pattern set but differ in post-pattern integration, which is where the drift in the defect above lives; (b) `api/chats.ts:85-109` reimplements Clerk `verifyToken` from scratch, duplicating `utils/auth.ts:23-58`, so a future auth fix will not propagate.
- **Complexity hotspots:** `components/v2/V2DraftingMagicPage.tsx` (2808 lines) and `api/_lib/agentLoop.ts` (1661 lines) each mix several concerns in one file; that is where the next hard-to-catch regression will come from.
- **Minor:** stale docstring at `api/_lib/sessionStore.ts:366` ("30s" vs `LOCK_TTL_SECONDS=330`); the regex-only server backstop by design cannot catch a bare single-word name that slips the browser detector (`services/sanitization/detectionPipeline.ts:565-572`) — an accepted boundary worth documenting for counsel, not a code bug.

## 2. Prioritized improvements

### P0 — Close the CA driver-license wire-redaction regression (T-PII-032)
- **What/why:** an attorney typing a client's CA driver-license number is a core confidentiality case; the wire pipeline silently misses it today (`runTrapsWire.mjs` 119/120). Evidence above.
- **Bet:** *if* an in-format CA DL (`[A-Z]\d{7}`, e.g. `A1234567`) passes through `detectPii()`, *then* it is tokenized before send, *observable:* `runTrapsWire.mjs` reports 120/120, 0 wire leaks, exit 0.
- **Invariants:** T-PII-032 tokenized (INV1); no other trap or `sanitization.test.mjs` case regresses (INV2); change confined to the on-device detection layer (INV3).
- **Gate intents:** `tsx tests/traps/runTrapsWire.mjs` exits 0 at 120/120; `runTraps.mjs` 100/100 and `sanitization.test.mjs` green; `git diff --name-only` ⊆ sanitization scope.
- **Effort:** S (bounded; most of the work is root-causing the suppression). **Authored as the contract in §3.**

### P1 — Add a CI test gate that runs the full local suite (including `runTrapsWire`)
- **What/why:** the absence of this gate is why P0 shipped. Every `tests/*.test.{mjs,ts}` and both trap runners are pure-local (no network) and run in seconds — there is no cost reason they are not gating merges.
- **Bet:** *if* a GitHub Action runs the whole local suite on every push/PR, *then* a red trap or unit test blocks merge, *observable:* the workflow fails CI when any test is intentionally broken and passes on a clean tree.
- **Invariants:** the aggregate command exits nonzero if any test/trap fails; no live-API script (`agent:smoke*`, `latency:baseline`) is in the CI path.
- **Gate intents:** a `yarn test` (or `test:all`) script exists and runs all 15 unit files + `runTraps.mjs` + `runTrapsWire.mjs`; the workflow file invokes it; a deliberately-broken test turns CI red.
- **Effort:** S (½ day). Wire `test:all` in `package.json`, add `.github/workflows/test.yml`.

### P1 — Make the two sanitizer pipelines provably agree
- **What/why:** the analyze vs wire drift is a recurring class of bug, not a one-off. A conformance test asserting `analyze()` and `detectPii()` produce the same category coverage over every trap fixture converts "keep them in sync by discipline" into a gate.
- **Bet:** *if* a conformance test runs both pipelines over `tests/traps/manifest-v1.json`, *then* any future divergence fails a test, *observable:* introducing a pattern into one pipeline but not the other turns the test red.
- **Invariants:** for every trap, the set of `must_redact` categories detected by `detectPii()` ⊇ those detected by `analyze()`; test is deterministic and offline.
- **Gate intents:** new conformance test exits 0 on the fixed tree and nonzero under an injected divergence.
- **Effort:** M (1–2 days).

### P2 — Consolidate `api/chats.ts` auth onto `httpGuard.requireUser`
- **Bet:** *if* `chats.ts` calls the shared guard, *then* removing the duplicated `verifyToken` (`chats.ts:85-109`) leaves behavior unchanged, *observable:* `routeSecurity`/auth tests stay green and `grep verifyToken api/chats.ts` is empty.
- **Invariants:** unauthenticated requests still 401; the local-dev bypass matches the rest of the app. **Effort:** S.

### P2 — Decompose the two complexity hotspots along concern seams
- `agentLoop.ts` (model-call / tool-dispatch / policy / audit) and `V2DraftingMagicPage.tsx` (upload / state / render / export). **Bet:** extract modules with unit tests behind each seam without behavior change; *observable:* build + all tests green, per-file line counts drop materially. **Effort:** L. Do this only when a feature forces you into these files.

### P2 — Alert on fail-open audit/manifest write failures
- The fail-open audit path (`api/_shared/auditLog.ts:107-150`) is correct for availability but, on a compliance product, a silently-dropped audit record should page someone. **Bet:** emit a structured error metric on write failure; *observable:* a forced Redis-write failure produces a monitored event. **Effort:** S.

## Context

This contract targets the repository at `/Users/arjundivecha/Dropbox/AAA Backup/A Working/California-Law-Chatbot`, a Clerk-authenticated California family-law research and drafting assistant. Before any client text leaves the browser it passes through an on-device sanitization pipeline that detects PII and replaces each span with an opaque token ("tokenize"); the raw values never travel to the model provider.

There are two sanitization code paths that share one regex pattern set:

- The **analyze pipeline** — `api/_shared/sanitization/index.ts` (`analyze()`), exercised by `tests/sanitization.test.mjs` and the 100-trap runner `tests/traps/runTraps.mjs` (npm: `yarn test:traps`).
- The **wire pipeline** — `services/sanitization/detectionPipeline.ts` (`detectPii()`), the code the browser actually runs before `fetch()`. It is exercised by the 120-trap runner `tests/traps/runTrapsWire.mjs`, which is **not wired to any npm script and not run in CI**.

Both import the same regex patterns via `runPatterns` from `api/_shared/sanitization/patterns.ts`. That file already defines `CA_DRIVER_LICENSE` (line ~58: one letter followed by seven digits), and `tests/sanitization.test.mjs` asserts `mustDetect(CA_DRIVER_LICENSE, ...)` — which passes. So the standalone regex is correct.

The defect is a pipeline-integration miss, not a missing pattern. Running `tsx tests/traps/runTrapsWire.mjs` today reports **119/120**, with `T-PII-032 [mixed_direct_pii] missed=driver_license:"A1234567"` (input: `Driver's license A1234567 belongs to my client; please confirm.`, defined in `tests/traps/manifest-v1.json` around line 921). The span the raw pattern would produce is being dropped somewhere in the fuller wire pipeline — candidate causes to investigate in the real repo: allowlist/stoplist suppression (`api/_shared/sanitization/allowlist.ts`, `services/sanitization/glinerPostProcess.ts`), span-overlap resolution ("more specific wins"), or an interaction with OPF/name spans. Commit `bc3677f` previously recorded a 120/120 trap pass, so this is a regression. The exact fix site is `DISCOVER_TARGETS`; the coding model must find where the driver_license span is suppressed and restore it without weakening other categories.

Terms: **tokenize** = replace a raw PII span with a reversible opaque token before send; **wire leak** = a raw PII value reaching the network; **trap** = a labelled fixture with a `must_redact` expectation.

## Build Loop vs Product Loop

The **build loop** is fully machine-verifiable here. Gate G1 proves the wire pipeline now tokenizes the T-PII-032 driver-license (120/120, 0 wire leaks). Gate G2 proves no regression: the 120 wire traps, the 100 analyze traps, and `tests/sanitization.test.mjs` all exit 0. Gate G3 proves the change stayed inside the on-device detection layer via `git diff` scope containment. When all three gates exit 0, the implementation contract is satisfied.

The **product loop** is the real-world bet: that no California driver-license number an attorney types is ever exposed to a third party un-redacted, across every workflow and input shape, in production. That is a confidentiality guarantee that can only be judged by live monitoring, adversarial red-teaming, and counsel review over many real messages — not by one fixture. The coding model may not claim the product bet is satisfied solely because the gates pass; green gates prove the specific regression is closed and no covered case regressed, not that the confidentiality guarantee holds universally.
